# PostgreSQL 多層級自動備份系統 - 使用說明書

## 📋 功能概述

本系統實現三層級自動備份策略：
- **10分鐘級備份**：頻繁備份，下一個小時同時段覆蓋
- **小時級備份**：每小時備份，第二天同時段覆蓋  
- **日級備份**：每日21:30備份，自動保留最近7天

---

## 🚀 安裝步驟

### 步驟1：準備檔案

你已經有以下3個備份腳本檔案：
- `backup_10min.sh` - 10分鐘備份腳本
- `backup_hourly.sh` - 每小時備份腳本
- `backup_daily_7day.sh` - 每日備份腳本（保留7天）

### 步驟2：上傳到Ubuntu伺服器

在Windows上使用WinSCP或其他SFTP工具，將3個.sh檔案上傳到Ubuntu的以下目錄：

```
/home/tom/letsoj/oj-postgres-db/
```

確保上傳後的檔案路徑為：
```
/home/tom/letsoj/oj-postgres-db/backup_10min.sh
/home/tom/letsoj/oj-postgres-db/backup_hourly.sh
/home/tom/letsoj/oj-postgres-db/backup_daily_7day.sh
```

### 步驟3：給予執行權限

在Ubuntu終端執行以下命令：

```bash
chmod +x /home/tom/letsoj/oj-postgres-db/backup_10min.sh
chmod +x /home/tom/letsoj/oj-postgres-db/backup_hourly.sh
chmod +x /home/tom/letsoj/oj-postgres-db/backup_daily_7day.sh
```

### 步驟4：配置Crontab

在Ubuntu終端執行：

```bash
crontab -e
```

編輯器會打開（通常是nano或vim），在檔案末尾加入以下內容：

```
# PostgreSQL 多層級自動備份任務
# 每10分鐘備份一次（下一個小時同時段會覆蓋）
*/10 * * * * /home/tom/letsoj/oj-postgres-db/backup_10min.sh

# 每小時第0分鐘備份一次（第二天同時段會覆蓋）
0 * * * * /home/tom/letsoj/oj-postgres-db/backup_hourly.sh

# 每天21:30備份一次（保留最近7天）
30 21 * * * /home/tom/letsoj/oj-postgres-db/backup_daily_7day.sh
```

**保存方式：**
- 如果是 **nano**：按 `Ctrl+O` 儲存，`Ctrl+X` 退出
- 如果是 **vim**：按 `Esc`，輸入 `:wq` 儲存退出

### 步驟5：驗證配置

執行以下命令確認crontab已正確設置：

```bash
crontab -l
```

應該會顯示類似以下內容：
```
*/10 * * * * /home/tom/letsoj/oj-postgres-db/backup_10min.sh
0 * * * * /home/tom/letsoj/oj-postgres-db/backup_hourly.sh
30 21 * * * /home/tom/letsoj/oj-postgres-db/backup_daily_7day.sh
```

### 檔案命名規則

| 備份類型 | 檔名範例 | 覆蓋規則 |
|---------|---------|--------|
| 10分鐘 | `postgres_10min_14_30.zip` | 每10分鐘更新，下一個小時同時段會覆蓋 |
| 每小時 | `postgres_hourly_14.zip` | 每小時第0分更新，第二天同時段會覆蓋 |
| 每日(7天) | `postgres_daily_2025-12-14.zip` | 每天21:30更新，超過7天自動刪除 |

**範例：**
- 14:30 的10分鐘備份：`postgres_10min_14_30.zip`
- 15:30 的10分鐘備份：會覆蓋上面那個檔案
- 2025年12月14日的備份：`postgres_daily_2025-12-14.zip`
- 2025年12月8日的備份：會被自動刪除（超過7天）

## 🔄 工作流程說明

### 10分鐘備份工作流程
```
14:00 → 建立 postgres_10min_14_00.zip
14:10 → 建立 postgres_10min_14_10.zip
14:20 → 建立 postgres_10min_14_20.zip
14:30 → 建立 postgres_10min_14_30.zip
...
15:00 → 建立 postgres_10min_15_00.zip
15:10 → 建立 postgres_10min_15_10.zip
...
15:30 → ✓ 覆蓋掉14:30的 postgres_10min_14_30.zip
```

### 每小時備份工作流程
```
2025-12-14 14:00 → 建立 postgres_hourly_14.zip
2025-12-14 15:00 → 建立 postgres_hourly_15.zip
...
2025-12-14 21:00 → 建立 postgres_hourly_21.zip
...
2025-12-15 14:00 → ✓ 覆蓋掉前一天的 postgres_hourly_14.zip
```

### 每日備份工作流程
```
2025-12-14 21:30 → 建立 postgres_daily_2025-12-14.zip
2025-12-15 21:30 → 建立 postgres_daily_2025-12-15.zip
...
2025-12-21 21:30 → 建立 postgres_daily_2025-12-21.zip
2025-12-22 21:30 → 建立 postgres_daily_2025-12-22.zip
              → 自動刪除 postgres_daily_2025-12-15.zip (超過7天)
```

## ⚙️ 故障排查

### 檢查備份是否正常執行

檢視crontab日誌：
```bash
grep CRON /var/log/syslog
```

或在Ubuntu 18.04+使用：
```bash
journalctl -u cron
```

### 手動測試備份腳本

執行備份指令測試是否正常運作：

```bash
/home/tom/letsoj/oj-postgres-db/backup_10min.sh
```

如果成功，應該會在備份目錄看到新的 `.zip` 檔案。

### 檢查備份目錄是否存在

確認備份目標目錄存在且可寫入：

```bash
ls -la /media/sf_05-05-oj-shared-with-ubuntu/postgres_backup/
```

若目錄不存在，需要先建立：

```bash
mkdir -p /media/sf_05-05-oj-shared-with-ubuntu/postgres_backup/
```

### 檢查磁碟空間

確保有足夠空間存放備份：

```bash
df -h /media/sf_05-05-oj-shared-with-ubuntu/
```

### 檢查PostgreSQL資料庫路徑

確認PostgreSQL資料庫確實在以下位置：

```bash
ls -la /home/tom/letsoj/oj-postgres-db/
```

若路徑不對，需要修改3個備份腳本中的 `DB_SOURCE` 變數。

## 🛠️ 進階設置

### 修改備份時間

編輯crontab：
```bash
crontab -e
```

**Crontab 時間格式解析：**
```
分鐘 小時 日期 月份 週幾 命令
  0    21    *    *    *   (代表每天21:00)
 30    21    *    *    *   (代表每天21:30)
*/10   *    *    *    *   (代表每10分鐘)
```

**常見例子：**
- 改為每15分鐘備份：`*/15 * * * * ...`
- 改為每2小時備份：`0 */2 * * * ...`
- 改為每天22:00備份：`0 22 * * * ...`

### 修改備份目錄

編輯對應的 `.sh` 檔案，修改 `BACKUP_DEST` 變數的值。

### 修改保留天數

編輯 `backup_daily_7day.sh`，修改最後一行：
```bash
find "$BACKUP_DEST" -name "postgres_daily_*.zip" -mtime +7 -delete
```

其中 `+7` 表示7天，改為 `+14` 即為14天。

## 📞 常見問題

**Q: 備份檔案為什麼沒有被建立？**  
A: 
1. 檢查crontab是否正確配置 (`crontab -l`)
2. 檢查 .sh 檔案是否有執行權限 (`ls -la backup_*.sh`)
3. 檢查備份目錄是否存在 (`ls /media/sf_05-05-oj-shared-with-ubuntu/postgres_backup/`)
4. 檢查PostgreSQL資料庫路徑是否正確

**Q: 如何手動執行備份？**  
A: 
```bash
/home/tom/letsoj/oj-postgres-db/backup_10min.sh
```

**Q: 如何刪除舊備份？**  
A: 
```bash
rm /media/sf_05-05-oj-shared-with-ubuntu/postgres_backup/postgres_10min_*.zip
```

**Q: 如何停用備份任務？**  
A: 
編輯crontab並註解掉對應行：
```bash
crontab -e
# 在要停用的行前面加上 #
```

**Q: 備份檔案太大怎麼辦？**  
A: 可以編輯 .sh 檔案，將 `zip -r` 改為使用 `tar` 或其他壓縮工具以提高壓縮率。

## 📝 備份腳本內容

### backup_10min.sh
```bash
#!/bin/bash
DB_SOURCE=/home/tom/letsoj/oj-postgres-db
BACKUP_DEST=/media/sf_05-05-oj-shared-with-ubuntu/postgres_backup
cd "$DB_SOURCE"
zip -r postgres.zip postgres
MINUTE=$(date +'%H_%M')
mv postgres.zip "$BACKUP_DEST/postgres_10min_${MINUTE}.zip"
```

### backup_hourly.sh
```bash
#!/bin/bash
DB_SOURCE=/home/tom/letsoj/oj-postgres-db
BACKUP_DEST=/media/sf_05-05-oj-shared-with-ubuntu/postgres_backup
cd "$DB_SOURCE"
zip -r postgres.zip postgres
HOUR=$(date +'%H')
mv postgres.zip "$BACKUP_DEST/postgres_hourly_${HOUR}.zip"
```

### backup_daily_7day.sh
```bash
#!/bin/bash
DB_SOURCE=/home/tom/letsoj/oj-postgres-db
BACKUP_DEST=/media/sf_05-05-oj-shared-with-ubuntu/postgres_backup
cd "$DB_SOURCE"
zip -r postgres.zip postgres
DATE=$(date +'%Y-%m-%d')
mv postgres.zip "$BACKUP_DEST/postgres_daily_${DATE}.zip"
find "$BACKUP_DEST" -name "postgres_daily_*.zip" -mtime +7 -delete
```
